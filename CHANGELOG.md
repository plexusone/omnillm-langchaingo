## [v0.1.0] - 2026-04-13

### Highlights

- LangChainGo adapter for omnillm - use any omnillm provider with LangChainGo chains, agents, and RAG pipelines

### Added

- `New()` function to create LangChainGo `llms.Model` from omnillm provider ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- Full `llms.Model` interface implementation with `GenerateContent()` and `Call()` methods ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- Streaming support with `llms.WithStreamingFunc()` callback option ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- Tool/function calling support with automatic conversion between LangChainGo and omnillm formats ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- All LangChainGo call options: temperature, max tokens, top-p, stop words, seed ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- Token usage tracking in `GenerationInfo` response metadata ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))
- Message role mapping between LangChainGo and omnillm (system, user, assistant, tool) ([`ae3886d`](https://github.com/plexusone/omnillm-langchaingo/commit/ae3886d))

### Fixed

- Add error handling for unsupported content types (ImageURLContent, BinaryContent) ([`e7f0f35`](https://github.com/plexusone/omnillm-langchaingo/commit/e7f0f35))
- Add default case in message conversion switch for unknown content types ([`e7f0f35`](https://github.com/plexusone/omnillm-langchaingo/commit/e7f0f35))

### Documentation

- README with installation, usage examples (omnillm, omnillm-core, chains, tools, streaming), and architecture diagram ([`c5d7ee8`](https://github.com/plexusone/omnillm-langchaingo/commit/c5d7ee8))

### Tests

- Unit tests for `New()`, `GenerateContent()`, `Call()`, role mapping, and tool conversion ([`becf925`](https://github.com/plexusone/omnillm-langchaingo/commit/becf925))
